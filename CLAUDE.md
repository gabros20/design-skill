# Claude Code repository guide

Read [AGENTS.md](AGENTS.md) before changing runtime behavior and
[CONTRIBUTING.md](CONTRIBUTING.md) before preparing a release. Keep `design` focused on experience
intent — research, structure, behavior, visual craft, content, systems, critique, and handoff — and
hand production frontend, architecture, marketing, quality audits, and product strategy to their
owning capabilities.

Preserve the canonical homes: motion values in `motion.md`, accessibility floors in
`accessibility.md`, the anti-default catalog in `visual-direction.md`. Run `scripts/check-sync`
after every runtime, routing, metadata, documentation, or evaluation change. Do not claim visual
completion unless the relevant rendered evidence was inspected.
